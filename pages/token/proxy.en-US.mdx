# OpenMask Token

## Proxy feature

The jetton wallet supports a proxy feature with allows a `proxy` message. Upon receiving a `proxy` message with a target address,
it should include jetton balance and jetton wallet owner to message and send it to the target address. Handling a proxy notification
allows us to verify that at the moment of the transaction, the sender owns some amount of jettons.

The implementation of this technology allows us to implement validation at a contract level and provide utility features for token holders.

The source code for [Jetton Wallet](https://github.com/OpenProduct/openmask-token/blob/main/contracts/jetton-wallet.fc)
and [Example Proxy Handler](https://github.com/OpenProduct/openmask-token/blob/main/contracts/proxy-handler-example.fc)
you could find in project [GitHub page](https://github.com/OpenProduct/openmask-token).
The GitHub project includes instructions and scripts to deploy the same jetton contracts.
